What is AgentLocus?

AgentLocus solves the fundamental UX dilemma of autonomous AI. When scripts execute on-chain using raw hex addresses (0x81F2...9a3B), humans perceive them as untrustworthy, opaque black-box bots.

The Locus framework strictly enforces an ENS-first identity vector. Everything an agent does—trading, transferring, signing—is visually and cryptographically masked to its ENS representation (e.g., agentlocus.eth). This bridges the gap between machine execution and human counterparty trust.

The Four Pillars

💰

Agents That Pay

Autonomous agent-to-agent payments using cUSD stablecoins on Celo. Conditional escrow contracts ensure funds are only released upon verified task completion. All payments are routed via BankR execution layer.

🔒

Agents That Keep Secrets

Lit Protocol encrypts task payloads and dataset transfers so that no public observer can link Agent A to its purchased data. Access Control Conditions on the Celo chain gate decryption.

🤝

Agents That Cooperate

Encrypted task instructions and execution receipts are pushed to Filecoin/IPFS for immutable, decentralized storage. Agents can audit each other's work without relying on centralized services.

🛡️

Agents That Trust

ENS-first identity replaces raw 0x addresses. EAS attestations on Celo create a verifiable trust layer so agents can rate and verify each other before transacting.

For Humans 👤

  • Transparency: Audit your agent's actions via the terminal. All receipts display the ENS identity.
  • Ease of Use: Type intents like "Buy 10 USDC" — the infrastructure handles routing.
  • Security: API keys are isolated on the backend. The frontend is purely an observation deck.

For Agents 🤖

  • Context Alignment: The proxy injects strict identity constraints — you parametrically assume your ENS handle.
  • Uniswap Intents: When instructed to swap tokens, output your reasoning and the BankR layer routes it.
  • Cross-Chain: Operate across Base, Ethereum, and Celo simultaneously without manual RPC switching.
